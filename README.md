# Pen Plotter Control Suite

Modern tooling tuned for the [LY Drawbot](https://nl.aliexpress.com/item/32967477326.html?gatewayAdapt=glo2nld) AliExpress pen plotter, built around GRBL control, servo-based pen lift, and four-corner bed compensation.

![Pen Plotter UI Placeholder](docs/images/ui.png)

## Overview

The Drawbot ships with a lightweight frame, a hobby servo that raises/lowers the pen, and a working area that is rarely perfectly level. This project wraps those quirks with:

- `nicegui_app.py` – Browser UI for calibrating bed corners, tuning pen lift speed and settle times, importing SVGs, running plot jobs with live progress/ETA, and issuing GRBL commands on the fly (with a built-in `$`-command reference).
- `pattern.py` / `penplot_helper.py` – Geometry primitives plus a renderer that streams optimized tool paths (global or tiled travel ordering, hatch-direction control) while honouring servo timing and reporting progress.
- `patterns/` – Generative notebooks and exported SVGs that double as a plotting portfolio.

Use the GUI to drive hardware, import SVG artwork, or call the backend directly when you script bespoke jobs.

## Quick Start

```bash
python -m venv .venv
source .venv/bin/activate  # Windows: .venv\Scripts\activate
pip install --upgrade pip
pip install -r requirements.txt
```

Notebook tooling (optional):

```bash
pip install -r requirements-notebooks.txt
```

Python 3.10+ is recommended. The app works without a connected plotter, so you can explore the UI before bringing hardware online.

## Run the GUI

```bash
python nicegui_app.py \
  --area-size 300x245
```

Open the printed `http://127.0.0.1:PORT` link in your browser.

**Typical workflow**
- Drop SVG files onto the `Load` tab to queue them for plotting — this is the default path for artwork generated by the notebooks. Imported artwork is flipped from SVG's Y-down space into the bed's Y-up space and re-centred automatically, so it matches what you see on screen.
- Alternatively, paste pattern text into the load field and set feed/pen overrides inline when iterating on custom geometry.
- Use the four corner inputs to level the not-so-flat drawbed; the app generates a bilinear compensation map so the pen keeps consistent pressure.
- Tune servo positions, lift speed, and wait times to match different pens or paper thicknesses.
- Configure the run (z-mode, feeds, travel optimization, hatch orientation) on the `Config` tab, then start the job from the `Run` tab and watch live progress.
- Explore the built-in terminal to jog the plotter, send raw G-code, and tweak GRBL configuration values without leaving the browser. The `?` button opens a clickable `$`-command reference so you don't have to memorise GRBL settings.

### Run tab — plotting & progress

- **Start / Pause / Stop** with predictable behaviour. Pause issues a GRBL feed-hold and resumes cleanly; Stop halts motion, flushes GRBL's planner buffer (soft reset), lifts the pen, and restores your work origin instead of leaving the machine stuck mid-buffer.
- The pen stays fully **up** from the moment you press Start until it reaches the first point of the first stroke — no dragging on the approach.
- A **progress bar** with percentage, **elapsed**, **estimated total**, and **remaining** time. The estimate starts from path length and feeds, then refines to a measured projection once underway.
- When the artwork uses **more than one pen colour**, the Run tab also shows an **estimated time per pen** breakdown (each colour costed as its own filtered run, plus an all-pens total) and highlights the currently selected pen. With a single colour this is hidden. Selecting a pen in the dropdown scopes both the plot and its ETA to that colour.

### Config tab — optimization & motion

- **Travel order**: `None`, `Global NN` (one nearest-neighbour pass over every stroke), or `Tiled`. Tiled splits the artwork into an *N×N* grid (set the grid size), nearest-neighbour optimises the strokes whose centre falls in each tile, and walks the tiles row by row — useful for keeping locally related work together rather than letting a global pass criss-cross the sheet.
- **Hatch orientation**: `Optimize` (strokes may be reversed to minimise travel), `Directional` (parallel lines are all drawn the same way — left→right, and bottom→top for vertical lines), or `Keep` (every stroke drawn in its original direction, never reversed).
- **Z-mode** controls how pen height tracks the bed compensation along a stroke:
  - `centroid` / `start` set one pen height per stroke — **smooth, no mid-line stops** (recommended for most work).
  - `per_segment` / `threshold` adjust height along the stroke. Because the pen lift is a servo on GRBL's spindle PWM (`M3 Sxxx`), each height change forces a planner sync (a brief stop) outside laser mode — see the note below. `threshold` only re-commands the servo when the height changes by more than the configured amount, and identical consecutive servo values are skipped, so it minimises those stops.

## Programmatic Use

The backend runs headless when you want to drive the LY Drawbot directly from Python:

```python
from pattern import Pattern, Circle, Renderer
from penplot_helper import GRBL, Config

pattern = Pattern().add(Circle((150.0, 120.0), r=80.0, pen_id=1))

grbl = GRBL(Config(port='/dev/tty.usbserial-A50285BI')).connect()
renderer = Renderer(grbl, z_mode='centroid')
renderer.run(
    pattern,
    optimize='tiled',        # None | 'nn' | 'tiled'
    tile_grid=4,             # N×N tiling when optimize='tiled'
    hatch_orient='optimize', # 'optimize' | 'directional' | 'keep'
    combine={'join_tol_mm': 0.05},
    resample={'max_dev_mm': 0.1, 'max_seg_mm': None},
)
```

While a run is in progress you can poll `renderer.progress_snapshot()` for the
fraction done, elapsed time and ETA, and call `request_pause()` /
`request_resume()` / `request_cancel()` to control it. Apply geometry helpers
such as `Pattern.optimize_order_nn`, `Pattern.optimize_order_tiled`,
`Pattern.apply_hatch_orientation`, `resample_polylines`, and `combine_endpoints`
to clean up geometry before execution.

## Troubleshooting

- macOS serial ports appear under `/dev/tty.usb*`; Linux typically uses `/dev/ttyUSB*`. Run `python -m serial.tools.list_ports` to discover devices.
- Re-run the four-corner calibration whenever the machine is moved—the compensation matrix relies on fresh measurements.
- The GUI sets `MPLCONFIGDIR` to `.matplotlib_cache/` to avoid permission issues; keep the directory writable.
- Always validate new tool paths with the pen lifted and keep the pause/cancel controls handy for untrusted SVGs.
- **Pen stutters when height changes mid-stroke?** That is expected: the pen lift is a hobby servo driven by GRBL's spindle PWM (`M3 Sxxx`), and outside laser mode (`$32=0`) GRBL stops motion to apply each spindle change. Use `centroid`/`start` z-mode for smooth, stop-free lines. Truly smooth in-stroke height changes would require wiring the pen lift to a real GRBL axis; laser mode (`$32=1`) allows dynamic `S` but zeros PWM during travel, which drops the servo on pen-up moves.
